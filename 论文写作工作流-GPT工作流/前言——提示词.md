  # 📘 论文前言写作工作流（Introduction Writing Workflow）

> **核心原则：写作顺序倒写（4 → 3 → 2 → 1），阅读顺序正常展示（1 → 2 → 3 → 4）。**

---

## 🛠 Step 0 — 写作准备阶段

在正式写前言之前，需要完成以下准备：

- **一句话研究问题：**  
  `本研究旨在解决 ______，核心问题是 ______。`
- **整理参考文献（10–15 篇高质量最新文献）**
- **列出论文贡献（2–3 条最清晰）**

---

## 🧩 Step 4 — 创新点与论文框架（先写）

> 明确“你做了什么”和“贡献是什么”，这是整篇前言的锚点。

### 🔑 写作结构

1. **一句话介绍方法/研究内容**（不解释原因）
2. **列出贡献点（2–3 条，越精炼越好）**
3. **给出论文结构安排**

### 🔤 常用句式模板

- **介绍方法：**
  - `本文提出了一种新的…方法，用于…`
  - `本研究设计了一种能够…的框架/模型…`

- **贡献表达关键词：**
  - `提出/设计/实现/验证/首次实现/改进/构建`

- **论文结构说明示例：**

---

## 🧩 Step 3 — 新方法引入与研究动机

> 解释你**为什么想到这个方法**，而不是介绍方法细节。

### 🔑 写作结构

1. 承接现有研究不足（来自 Step 2）
2. 提出解决思路/核心理念
3. 说明潜在优势和合理性
4. 过渡到贡献总结（引向 Step 4）

### 🔤 常用表达：

- `为克服上述不足，本文提出…`
- `该方法的核心思想是利用…以实现…`
- `与现有方法相比，本研究具有…方面优势`
- `因此，本文将重点解决…并提出以下贡献。`

---

## 🧩 Step 2 — 相关工作总结与评述（有目的文献综述）

> 不是写百科式综述，而是**建立逻辑链证明：别人做过，但没解决干净，所以轮到你。**

### 🔑 写作结构：

1. 方法分类或发展脉络概述（可选但加分）
2. 逐类评价（优点→不足→研究空白）
3. 引导至你的研究动机（承接 Step 3）

### 🔤 常用逻辑衔接词：

| 逻辑功能 | 推荐短语              |     |
| ---- | ----------------- | --- |
| 补充   | `此外、同时、进一步来说`     |     |
| 对比   | `相比之下、然而、但与之不同的是` |     |
| 并行   | `与此并行的是…`         |     |
| 递进   | `随后、近年来、基于此`      |     |
| 转折不足 | `然而、但仍然、尽管如此`     |     |
| 举例   | `例如、以…为例、如相关研究所示` |     |

## 🧩 Step 1 — 研究问题提出与研究意义

> 目的：让读者相信“这个问题值得研究”。

### 🔑 写作结构：

1. 研究背景（宏观 → 领域 → 具体问题）
2. 问题由来与现阶段重要性
3. 研究意义（科学意义 / 工程价值 / 应用意义）
4. 挑战或困难（引到 Step 2）

### 🔤 常用句式：

- 背景描述：
  - `随着…快速发展，…问题变得愈发重要。`
  - `近年来，…逐渐成为研究热点。`
- 重要性：
  - `该问题的解决有助于… / 对…具有重要意义。`
- 挑战描述：
  - `然而，由于…该问题仍未被有效解决。`

---

## 📚 引文引用原则

- 推荐引用数量：**10–15 篇**
- 引用策略：**有目的引用，而不是堆砌**
- 原则：**先肯定 → 再反馈不足 → 引出动机**

---

## 📦 最终检查清单 Checklist

| 检查项                     | 是否完成 |
| ----------------------- | ---- |
| 是否一句话总结研究问题             |      |
| 是否构建“背景→现状→不足→动机→创新点”闭环 |      |
| 贡献是否明确、可数、可验证           |      |
| 文献引用是否最新、相关性高           |      |
| 逻辑衔接是否自然、无跳跃            |      |
